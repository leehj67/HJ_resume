Keyword Ensemble Metadata for Retrieval Preprocessing in RAG Systems

RAG 시스템에서 문서 전처리를 위한 키워드 기반 메타데이터와 BM25 융합 검색 신호 연구

🔗 GitHub 저장소

Research Motivation

Retrieval-Augmented Generation (RAG) 시스템은 검색 단계의 품질에 크게 의존합니다. 그러나 실제 문서 컬렉션에는 비정형 텍스트가 많아, BM25 같은 전통적인 sparse retrieval 모델이 의미적 관련성을 충분히 포착하지 못할 수 있습니다.

키워드 기반 문서 메타데이터는 경량 전처리 방식으로, 의미적 커버리지를 확장하고 후보 문서 선택을 개선하는 데 도움을 줍니다.

  • 키워드 앙상블 기반 문서 메타데이터 생성
  • 키워드 중첩 신호를 이용한 융합 검색
  • 키워드 커버리지와 검색 순위 성능 간 관계 분석

Key Idea

문서에 키워드 추출 알고리즘을 적용하고, 추출된 키워드를 keyword ensemble metadata set으로 결합해 추가 검색 신호로 사용합니다.

융합 전략으로 검색 점수를 계산합니다:

score = (1 - α) × BM25 + α × keyword_overlap

여기서 keyword_overlap = | query_tokens ∩ document_keywords | 입니다.

이를 통해 키워드 메타데이터가 경량 보조 신호로 검색에 기여합니다.

Experiments

BEIR 벤치마크 데이터셋: SciFact, NFCorpus, Arguana

키워드 추출 방법

  • RAKE
  • YAKE
  • Ensemble (RAKE + YAKE)

검색 모델

  • BM25 baseline
  • BM25 + keyword fusion

평가 지표

  • NDCG@10
  • MRR
  • Recall@10

추가 실험: Alpha sweep, 키워드 커버리지 분석, Coverage vs retrieval trade-off

Results

Alpha Sweep Analysis

융합 가중치 α가 검색 성능에 미치는 영향. 작은 키워드 가중치(α ≈ 0.05–0.1)가 BM25 순위와 키워드 신호 간 최적 균형을 제공합니다.

Coverage vs Retrieval Trade-off

키워드 앙상블은 문서 키워드 커버리지를 크게 늘리지만, 높은 커버리지가 항상 검색 순위 성능 향상으로 이어지지는 않습니다. 키워드 메타데이터는 주요 순위 신호보다는 약한 보조 검색 신호로 동작합니다.

Main Findings

  • 키워드 앙상블은 문서 키워드 커버리지를 증가시킵니다.
  • 커버리지 증가가 항상 검색 순위 개선을 의미하지는 않습니다.
  • 키워드 신호는 BM25와 융합 시 보조 신호로 동작합니다.

키워드 기반 메타데이터는 후보 필터링, 문서 전처리, RAG 파이프라인 내 검색 신호 보강에 더 유용할 수 있습니다.

Quick Start

pip install rank-bm25 datasets beir rake-nltk yake python run_benchmark.py

프레젠테이션 모드 (SciFact, NFCorpus, Arguana):

python -m meta.benchmark.run_benchmark --presentation --force-reset

Alpha sweep fusion 실험:

python -m meta.benchmark.run_benchmark --alpha-sweep-fusion
